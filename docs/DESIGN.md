# Agent Pets 设计规约 / Design Conventions

本文件收录插件渲染与交互的设计规约,新增背景 / 宠物 / 布局时请遵循,以保证跨主题的一致性与可读性。所有坐标基于 **144 × 144** 的按键画布。

关键常量集中在:
- `plugin/actions/Pet.js` —— 布局与地面基线、Agent 图标、遮罩。
- `plugin/actions/backgrounds.js` —— 背景层与地平线规范。
- `plugin/actions/petArt.js` —— 宠物美术与着地点。

---

## 1. 画布分区与文字安全区

画布自上而下分为「信息带」和「Pet 区」:

| 区域 | Y 范围 | 用途 |
| --- | --- | --- |
| 信息带(文字安全区) | `0 ~ 84`(`SCRIM_H`) | 表头(Agent 图标 / session id / 本轮时长)、问题行、反馈行;顶部有渐变遮罩压暗 |
| Pet 区 | `84 ~ 144` | 宠物本体、背景主体物件、地面 |

**规约**:主要实心物件(房子 / 狗窝 / 篱笆 / 太阳等)必须落在 Pet 区(`y ≳ PET_TOP = 84`),避免压住文字。天空渐变、粒子等低对比氛围元素可铺满整块。

## 2. 地面基线(宠物着地)

- 常量 `GROUND_Y = 132`(`Pet.js`)。
- 宠物中心 `cy = GROUND_Y - Art.footOffset(species, growth, unit)`,即**脚(地面阴影)固定落在 `GROUND_Y`**。
- 因此小体型(蛋)不会悬空;体型变大时**向上生长**,头部进入信息带时会被遮罩压暗、文字覆盖其上,不影响可读性。
- `Art.footOffset()` 定义中心到着地点的距离:蛋 = `unit*0.9*1.05`;生物 = `unit*shape.h*0.95`(与地面阴影一致)。新增物种/形态时需保证该函数返回其真实底部偏移。

## 3. 地平线规范

带地平线的背景(草原 / 篱笆 / 花园 …):

- 使用 `horizonY(h) = PET_TOP + (h - PET_TOP) * 2/3`(≈ `y124`),即地平线固定在 **Pet 区自顶向下 2/3 处**。
- 目的:明显低于文字区边界(`PET_TOP`),**避免地平线与文字区边界重合**。
- 地面物件(篱笆、山丘、灌木)自地平线向上延伸;太阳 / 云等置于地平线以上的天空,且不进入文字安全区。

## 4. 背景层系统

每个主题内部分两层,一次选择、无需分别设置:

- `back(ctx, {phase,w,h})` —— 宠物**之后**绘制(天空 / 星空 / 网格 …)。
- `front(ctx, {phase,w,h})` —— 宠物之后、UI 文字**之前**绘制(雨 / 雪 / 花瓣 / 边框 …),盖在宠物身上。
- 可为**动态**(用 `phase` 驱动动画)或**静态**。
- 背景在「没有宠物」时也照常显示;有宠物时与宠物一同显示。

### 背景随机分配(拖入新按键时)

在「同屏其它按键未使用」的背景集合里**随机**选取,保证同屏尽量不重复;若该集合为空(按键数超过主题数,无法避免重复),再从全部主题里**完整随机**。实现见 `app.js` 的 `randomBg()`。

## 5. Agent 品牌图标

表头以品牌色圆角方块 + 白色简笔标记表示 Agent 类型(参考各家真实 logo,按 14px 显示尺寸简化):

| Agent | 主色 | 标记 |
| --- | --- | --- |
| Codex | `#10a37f`(OpenAI 绿) | 六瓣花结(中心镂空) |
| Claude | `#d97757`(Anthropic 陶土橙) | 放射星芒 |
| Qoder | `#8b5cf6`(官方紫) | 字母 `Q` |
| Pi | `#ec6a5e`(珊瑚) | 字母 `π` |

实现见 `Pet.js` 的 `AGENT_COLORS` 与 `drawAgentIcon()`。

## 6. 表头布局

- Agent 图标在最左(`MARGIN_L`),session id 紧随其后并**固定占位宽度**(超出裁剪)。
- 本轮时长在 session id 右侧的**剩余空间居中**,避免与 session id 重叠。

## 7. 本轮时长语义

时间显示为**「本轮对话的持续时间」**:
- 运行中 / 暂停:`now - turnStart`(turnStart = 我对 Agent 说话的时刻)。
- 完成:冻结为 `turnFrozenMs`(= Agent 最终回复时刻 − turnStart),完成后不再变化。

## 8. 成长与协调器(Demo)

- 全局单例 `PetCoordinator` 维护「宠物池」,每个按键只保存一个 `slot` 索引,渲染时按 slot 取 meta。
- Demo 生成节奏:随占用槽位数**指数增长后随机** —— `n` 只时取 `random(0, 10 * 2^n)` 秒(0→0-10s,1→0-20s,2→0-40s…),越拥挤越慢。见 `demo.js` 的 `spawnRandInterval()`。
- 成长由累计 token(Demo 下按运行时长估算)驱动,阶段阈值近似几何级递增,见 `stages.js`。
